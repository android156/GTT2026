Давай переделаем по уму блок документы. Документы желательно разбить по типам и добавить фильтр по типам. Сейчас есть типы Прайс-лист, Сертификат, Инструкции(переименовать в "Инструкции и технические описания"), Другое. Добавить тип "Учредительные документы и реквизиты". Сделать возможность добавлять типы. Для типов сделать чекбокс "Имеет свою страницу" по умолчанию, не проставлен. Для "Инструкции и технические описания" такой переключатель включен. Для таких типов переход по ссылке на документ должен приводить к переходу на отдельную страницу документа, где его уже в свою очередь можно будет скачать. Для типов, где "Имеет свою страницу" выключен - отдельная страница не нужна. Сразу скачивание. По всем документам сделать поиск по названию и краткому описанию. Эти поля нужно добавить. Как и поле справа "дата добавления".  Для названия сделать поле slug(заполняется автоматически транслитерацией, если не заполнено руками), чтобы документ скачивался (для типа "Инструкции и технические описания" открывалась страница) не по пути /documentation/static/uploads/documents/1.pdf/, а по пути /documentation/documents/slug/. При заполнении или смене slug (расширение в зависимости от типа файла) имя файла документа должно переименовываться в slug расширение не меняется и остается, как у оригинала. И ссылки соответственно должны меняться. Для документов изображений подключить наш инструментарий, как в галереях для редактирования и seo оптимизации.


Подробнее про отдельные страницы для типов где включен чекбокс "Имеет свою страницу". В админке такие страницы должны настраиваться отдельно не в общей /admin/documents/ , а в подменю с названием такого типа. Документы для таких типов должны загружаться на этом уровне админки.  Для pdf документов этого типа должна быть возможность править внутренние мета поля (по аналогии с seo полями изображений ) при первом сохранении заполнить внутренние мета-поля внутренний title - сюда записываем наше название, а в description или topic наше краткое описание. При редактировании этих полей в дальнейшем, сохраняем то, что введено руками.

Типовая страница документа типа, с включенным чекбоксом "Имеет свою страницу"

Заголовок h1

seo-text содержащий ссылку на скачивание документа и ссылки на связанные разделы нашего сайта. 

ссылка может включать изображение первой странички документа, по умолчанию пусть будет картинка заглушка.

В админке таких страниц должны настраиваться SEO-поля:

Title

Пример: Изопрофлекс 95 — техническое описание | Документация

H1

Пример: Изопрофлекс 95 — техническое описание

Description

Пример: Техническое описание труб Изопрофлекс 95 (0,6 МПа): типоразмеры, конструкция, 
пресс- и компрессионные фитинги, комплектующие и рекомендации по монтажу.
